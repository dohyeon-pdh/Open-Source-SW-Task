# 리눅스 프로세스 관련 명령어 정리 (`top`, `ps`, `jobs`, `kill`)
- 리눅스/유닉스 환경에서 **프로세스 상태를 확인하고 제어**할 때 자주 사용하는 명령어들입니다.  
- 각 명령어의 개념, 주요 옵션, 사용 예시를 정리해보았습니다.

----------------------------------------------------------

## 1. `top` : 실시간 프로세스/자원 모니터링

# 1.1 개념
- 시스템에서 **실행 중인 프로세스 목록과 CPU, 메모리 사용량** 등을 **실시간으로** 보여주는 모니터링 도구
- 윈도우의 작업 관리자(Task Manager)에 해당하는 텍스트 기반 툴이라고 생각하면 됩니다.

# 1.3 주요 단축
- top 실행 후 키보드 입력으로 동작을 제어할 수 있습니다.
q : top 종료
P : CPU 사용량 기준으로 정렬
M : 메모리 사용량 기준으로 정렬
T : 실행 시간 기준 정렬
k : 프로세스 종료(= 내부적으로 kill 사용)
h : 도움말(help)

# 1.4 자주 쓰는 옵션
- top -u 사용자이름   # 특정 사용자 프로세스만 보기
- top -p PID         # 특정 PID 프로세스만 모니터링

----------------------------------------------------------
## 2. ps : 현재 프로세스 상태 출력

# 2.1. 개념
- 현재 실행 중인 프로세스의 스냅샷(정적인 목록)을 출력합니다.
- top은 실시간 갱신, ps는 명령어를 실행한 순간의 상태만 보여줍니다.

# 2.2. 기본 사용 예시
ps                # 현재 쉘에서 실행 중인 프로세스만 출력
ps -e             # 시스템 전체 프로세스 출력
ps aux            # BSD 스타일, 거의 표준처럼 많이 쓰인다
ps -ef            # SysV 스타일, 많이 쓰이는 형식

# 2.3. 자주 쓰는 옵션 설명
a : 터미널과 연관된 모든 사용자의 프로세스
u : 프로세스의 소유자 정보(사용자) 포함
x : 터미널에 종속되지 않은 프로세스까지 포함
-e 또는 -A : 시스템의 모든 프로세스 출력
-f : full format (PPID, 시작 시간 등 자세한 정보 출력)

# 2.4. 자주 사용하는 패턴
* 전체 프로세스들 보기
=> ps aux
* 특정 프로세스만 이름으로 검색 (grep과 함께 사용)
=> ps aux | grep python
* 특정 PID의 상세 정보 확인
=> ps -p 1234 -f

----------------------------------------------------------
## 3. jobs : 현재 셸의 백그라운드 작업 목록

# 3.1. 개념
- 현재 셸에서 실행한 작업들 중 백그라운드/중지된 작업을 보여주는 명령어.
- 시스템 전체가 아닌, 현재 터미널/셸 세션 기준.
- &, Ctrl+Z, bg, fg 와 함께 자주 사용됩니다.

# 3.2. 관련 명령어 (bg, fg)
- 현재/지정 작업을 백그라운드로 실행
bg %1      # 1번 job을 background로 재개
- 현재/지정 작업을 포그라운드로 가져오기
fg %1      # 1번 job을 foreground로 가져오기

----------------------------------------------------------
## 4. kill : 프로세스에 시그널 보내기 (종료 등)
# 4.1. 개념
- kill은 말 그대로 프로세스를 죽이는 명령어라고 많이 알고 있지만, 실제로는 프로세스에 "시그널(signal)"을 보내는 명령어입니다.
- 가장 많이 쓰이는 시그널이 SIGTERM, SIGKILL 등 “종료 관련” 시그널이기 때문에 종료 용도로 쓰이는 것처럼 느껴질 수 있습니다.

# 4.2. 시그널 종류 확인
kill -l      # 사용 가능한 시그널 목록 출력
=> 출력 예시(일부): 1) SIGHUP   2) SIGINT   3) SIGQUIT  9) SIGKILL  15) SIGTERM ...
**자주 사용하는 시그널**
SIGTERM(15) : 정상 종료 요청. 프로세스가 정리할 시간 제공
SIGKILL(9) : 강제 종료. 정리 없이 즉시 종료, 마지막 수단
SIGINT(2) : 보통 Ctrl+C에 해당 (인터럽트)
SIGHUP(1) : 터미널 종료/설정 변경 등 알림

# 4.3. jobs와 함께 쓰는 kill
- jobs에서 job 번호를 보고 kill을 사용할 수도 있습니다.
jobs              # job 목록 확인
kill %1           # 1번 job에 SIGTERM 보내기
kill -9 %1        # 1번 job 강제 종료
----------------------------------------------------------

# (`top`, `ps`, `jobs`, `kill`) 네 명령어의 관계 정리 
1. top
→ 실시간으로 전체 시스템 프로세스와 자원 사용량을 확인
2. ps
→ 명령을 실행한 그 순간의 프로세스 목록을 스냅샷처럼 확인
3. jobs
→ **현재 셸에서 실행한 작업들(백그라운드/중지된 것)**의 목록 확인
4. kill
→ top, ps, jobs 등을 통해 찾은 특정 프로세스/작업에 시그널을 보내 종료 또는 제어
